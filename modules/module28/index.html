Weekly/monthly reviews. Threshold adjustments, model updates, fairness audits. AI Product Manager + Risk Committee. Triggered: monthly cycle + incident reports.

Quarterly governance review. Policy changes, major model decisions, regulatory response. Board + C-Suite + External Ethics Advisor. Triggered: quarterly + regulatory inquiry.

from typing import List, Dict, Optional
from enum import Enum

class RiskLevel(Enum):
    LOW      = "LOW"
    MEDIUM   = "MEDIUM"
    HIGH     = "HIGH"
    CRITICAL = "CRITICAL"

@dataclass
class GovernancePolicy:
    policy_id: str
    name: str
    threshold: float
    metric: str
    risk_level: RiskLevel
    action_required: str
    escalation_path: List[str]   # ["AI Engineer", "Risk Manager", "CRO"]
    review_frequency_days: int

        review_frequency_days=30

class PolicyEngine:
    def evaluate(self, metrics: Dict[str, float]) -> List[dict]:
        """Check all policies against current metrics."""
        violations = []
        for policy in LOANBOT_GOVERNANCE_CHARTER:
            value = metrics.get(policy.metric)
            if value is None: continue

            # Check threshold direction
            violated = (
                (policy.metric == 'approval_rate_ratio' and value < policy.threshold) or
                (policy.metric != 'approval_rate_ratio' and value > policy.threshold)
            ) if policy.metric != 'explanation_coverage' else value < policy.threshold

            if violated:
                violations.append({
                    'policy_id': policy.policy_id,
                    'name': policy.name,
                    'value': value,
                    'threshold': policy.threshold,
                    'risk_level': policy.risk_level.value,
                    'action': policy.action_required,
                    'escalate_to': policy.escalation_path[-1]
                })
        return violations

Bài 2: Bias Detection & Fairness Auditing

🎯 Sau bài này, bạn sẽ:

  • Implement 4 fairness metrics: Demographic Parity, Equalized Odds, Calibration, Individual Fairness
  • Detect disparate impact theo EU AI Act và US Equal Credit Opportunity Act (80% rule)
  • Build automated fairness audit report — chạy hàng tuần, alert khi threshold breach
  • Phân biệt bias trong data vs bias trong model vs bias trong thresholds
📌 Khái niệm: 4 Fairness Metrics

1. Demographic Parity: P(APPROVE | group A) ≈ P(APPROVE | group B). Không phân biệt nhóm trong approval rate. 2. Equalized Odds: TPR và FPR phải bằng nhau giữa các groups. Nếu Group A có 70% true positives, Group B cũng phải có ~70%. 3. Calibration: Khi model nói "70% likely to repay", 70% thực sự repay — across all demographic groups. 4. Individual Fairness: Hai cá nhân giống nhau (same credit profile) phải nhận cùng quyết định bất kể group membership. Nota bene: không thể satisfy tất cả 4 metrics cùng lúc khi base rates khác nhau giữa groups (Impossibility Theorem).

⚠️ Root Cause: Proxy Discrimination trong Credit Score

Rural bias không phải do model dùng "location" làm feature — FinTech Corp không collect location. Vấn đề sâu hơn: Rural borrowers có average employment_months thấp hơn (seasonal work) và lower income stability. Model học correlation này từ historical data — nhưng correlation này là artifact của historical underinvestment trong rural banking, không phải indicator of creditworthiness. Đây là proxy discrimination: model không dùng protected attribute trực tiếp nhưng dùng proxy variable correlates strongly với nó. Fix: (1) Separate employment tenure thresholds cho seasonal vs permanent employment; (2) Weight income consistency differently cho seasonal workers; (3) Consider alternative data: utility payments, mobile money history.

from dataclasses import dataclass
from typing import Dict, List, Tuple
import statistics

@dataclass
class FairnessReport:
    group: str
    n_cases: int
    approval_rate: float
    disparity_ratio: float   # vs reference group
    tpr: float               # True Positive Rate
    fpr: float               # False Positive Rate
    passes_80_rule: bool
    equalized_odds_gap: float

class FairnessAuditor:
    DISPARITY_THRESHOLD = 0.80   # 80% rule
    EQUALIZED_ODDS_TOL  = 0.05   # Max 5% TPR gap

    def compute_group_metrics(self, cases: list, group_attr: str,
                               group_val, reference_val) -> Tuple[FairnessReport, FairnessReport]:
        """Compute fairness metrics for a group vs reference group."""
        group_cases = [c for c in cases if getattr(c, group_attr) == group_val]
        ref_cases   = [c for c in cases if getattr(c, group_attr) == reference_val]

        def metrics(cases):
            if not cases: return 0, 0, 0
            approved = [c for c in cases if c.agent_decision == 'APPROVE']
            approval_rate = len(approved) / len(cases)
            with_outcome = [c for c in approved if c.outcome_30d]
            tp = sum(1 for c in with_outcome if c.outcome_30d == 'REPAID')
            fp = sum(1 for c in with_outcome if c.outcome_30d == 'DEFAULTED')
            tpr = tp / len(with_outcome) if with_outcome else 0
            fpr = fp / len(with_outcome) if with_outcome else 0
            return approval_rate, tpr, fpr

        g_ar, g_tpr, g_fpr = metrics(group_cases)
        r_ar, r_tpr, r_fpr = metrics(ref_cases)
        disparity = g_ar / r_ar if r_ar > 0 else 0

        group_report = FairnessReport(
            group=str(group_val), n_cases=len(group_cases),
            approval_rate=g_ar, disparity_ratio=round(disparity, 3),
            tpr=g_tpr, fpr=g_fpr,
            passes_80_rule=disparity >= self.DISPARITY_THRESHOLD,
            equalized_odds_gap=abs(g_tpr - r_tpr)

        return group_report

🧠 Kiểm tra nhanh

Fairness Impossibility Theorem nói rằng không thể satisfy cả Demographic Parity VÀ Equalized Odds cùng lúc khi base rates khác nhau giữa groups. Điều này có nghĩa là gì với FinTech Corp?

Gợi ý: Nghĩa là FinTech Corp phải explicitly choose ưu tiên metric nào — không thể have it all. Lựa chọn điển hình trong credit: (1) Equalized Odds (equal TPR/FPR) thường được prioritize vì nó ensure "equally qualified borrowers từ mọi group đều được approve ở rate tương đương" — fair process. (2) Demographic Parity đòi hỏi equal approval rates, điều này có thể require approving some unqualified applicants từ disadvantaged group để match rates. FinTech Corp cần document decision trong Governance Charter và get sign-off từ Ethics Committee + Legal. Không có "đúng" tuyệt đối — chỉ có trade-offs cần transparent.

Bài 3: Explainability & XAI — Giải Thích Quyết Định

🎯 Sau bài này, bạn sẽ:

  • Generate Natural Language Explanation (NLE) — giải thích bằng tiếng người thật cho khách hàng
  • Build counterfactual explanation: "Bạn cần thay đổi gì để được approve?"
  • Đáp ứng EU AI Act Art. 86: "Right to Explanation" cho automated high-risk decisions
💡 Analogy: Bác Sĩ vs Máy Chẩn Đoán

Máy chẩn đoán AI có thể nói "bệnh nhân này có 87% khả năng bị bệnh X" nhưng không thể nói TẠI SAO. Bác sĩ không thể ký kết bệnh án chỉ dựa trên con số — họ cần hiểu reasoning. Tương tự, NHNN không chấp nhận "LoanBot reject vì confidence 34%" — họ cần explanation cụ thể. Luật GDPR Art. 22 và EU AI Act Art. 86 trao cho người dùng quyền yêu cầu giải thích cho bất kỳ automated decision nào ảnh hưởng đến họ. Explainability không phải nice-to-have — là legal requirement.

Hình 1: XAI Pipeline cho FC-2024-004. 3 loại explanation: Feature Importance (employment là driver chính), Counterfactual (nếu emp≥12mo hoặc DTI≤0.38 → APPROVE), và Natural Language Explanation bằng tiếng Việt với actionable steps cho khách hàng.
Python (Counterfactual Explanation Generator) 
from dataclasses import dataclass
from typing import List
import anthropic

@dataclass
class Counterfactual:
    feature: str
    current_value: float
    required_value: float
    change_description: str
    feasibility: str   # "EASY" / "MODERATE" / "HARD"

class LoanBotExplainer:
    def __init__(self, client: anthropic.Anthropic, threshold_config):
        self.client = client
        self.cfg = threshold_config

    def counterfactual_explanation(self, app: dict) -> List[Counterfactual]:
        """Find minimum changes to flip decision to APPROVE."""
        cfs = []

        # Employment change
        if app['employment_months'] < self.cfg.min_emp_months:
            cfs.append(Counterfactual(
                feature='employment_months',
                current_value=app['employment_months'],
                required_value=self.cfg.min_emp_months,
                change_description=f"Tăng thời gian làm việc từ {app['employment_months']} tháng lên {self.cfg.min_emp_months} tháng",
                feasibility="MODERATE"  # Requires waiting
            ))

        # DTI change
        if app['dti_ratio'] > self.cfg.max_dti:
            required_income = app['monthly_debt'] / self.cfg.max_dti
            cfs.append(Counterfactual(
                feature='dti_ratio',
                current_value=app['dti_ratio'],
                required_value=self.cfg.max_dti,
                change_description=f"Giảm DTI từ {app['dti_ratio']:.2f} xuống {self.cfg.max_dti:.2f} bằng cách trả bớt nợ hiện tại hoặc tăng thu nhập",
                feasibility="HARD"
            ))

        # Alternative: provide collateral (bypass employment requirement)
        cfs.append(Counterfactual(
            feature='collateral',
            current_value=0,
            required_value=1,
            change_description="Cung cấp 1 tài sản đảm bảo (có thể bypass yêu cầu thời gian công tác)",
            feasibility="EASY"  # If customer has assets
        ))
        return sorted(cfs, key=lambda x: {'EASY': 0, 'MODERATE': 1, 'HARD': 2}[x.feasibility])

    async def natural_language_explanation(self, app: dict, decision: str,
                                                cfs: List[Counterfactual]) -> str:
        """Generate customer-facing Vietnamese explanation."""
        cf_text = "\n".join([f"- {c.change_description} [{c.feasibility}]" for c in cfs])
        resp = self.client.messages.create(
            model="claude-haiku-4-5-20251001", max_tokens=400,
            messages=[{"role": "user", "content":
                f"""Viết thư giải thích quyết định tín dụng bằng tiếng Việt lịch sự, ngắn gọn (150 từ).
Decision: {decision} | App ID: {app['id']}
Reasons: Employment {app['employment_months']}mo < required 12mo
Possible fixes:\n{cf_text}
Yêu cầu: Không kỹ thuật, thân thiện, nêu rõ bước tiếp theo."""}]
        )
        return resp.content[0].text

📌 Khái niệm: EU AI Act High-Risk Classification

EU AI Act (hiệu lực 2024) classify Credit Scoring AI là High-Risk (Annex III, Category 5b). Requirements cho High-Risk AI: (1) Risk Management System — documented và maintained; (2) Data Governance — training data quality; (3) Technical Documentation — model cards, performance metrics; (4) Logging — automatic logging của mỗi decision; (5) Human Oversight — không fully autonomous; (6) Accuracy, Robustness, Cybersecurity. FinTech Corp bán services cho EU customers → phải comply dù headquartered ở Việt Nam (extraterritorial scope). NHNN Circular 12/2020 có parallel requirements cho domestic market.

            })

        return {

            'generated_at': datetime.now().isoformat(),

Bài 5: Ethics Review Board — Process & Escalation

🎯 Sau bài này, bạn sẽ:

  • Cấu trúc AI Ethics Review Board: thành phần, frequency, và authority
  • Define escalation triggers: khi nào AI quyết định cần human review bắt buộc
  • Implement Ethics Incident Response: từ detection → investigation → remediation → prevention
  • Document AI ethics decisions: tạo paper trail cho regulatory và public accountability
🔑 Key Insight: Ethics Review Board vs Compliance Team

Compliance Team đảm bảo không vi phạm rules — reactive, checkbox-focused. Ethics Review Board asks deeper questions: Làm đúng luật nhưng có đúng về mặt đạo đức không? Ví dụ: LoanBot có thể legally reject rural borrowers nếu DTI cao — nhưng Ethics Board hỏi: "DTI cao có phải vì rural banking historically underserved không? Có alternative data sources nào fair hơn không?" Ethics Board có advisory power, không có veto — nhưng recommendations được documented, và ignoring Ethics Board recommendation cần explicit C-suite justification in writing.

Python (Ethics Escalation Engine) 
from dataclasses import dataclass
from enum import Enum
from typing import List, Optional

class EscalationLevel(Enum):
    NONE        = 0
    LOG_ONLY    = 1
    ALERT       = 2
    HUMAN_REVIEW = 3
    ETHICS_BOARD = 4
    HALT        = 5

@dataclass
class EscalationDecision:
    level: EscalationLevel
    triggers: List[str]
    required_action: str
    timeline_hours: int
    escalate_to: str

class EthicsEscalationEngine:
    """Determines when AI decision needs human ethics review."""

    def evaluate(self, decision: dict, app: dict,
                   fairness_metrics: dict) -> EscalationDecision:
        triggers = []
        max_level = EscalationLevel.NONE

        # HALT: proven bias violation + media exposure
        if fairness_metrics.get('disparity_ratio', 1.0) < 0.60:
            triggers.append('Severe disparate impact (<60% rule)')
            max_level = EscalationLevel.HALT

        # ETHICS_BOARD: bias detected + regulatory inquiry
        if fairness_metrics.get('nhnn_inquiry', False):
            triggers.append('Active NHNN regulatory inquiry')
            max_level = max(max_level, EscalationLevel.ETHICS_BOARD, key=lambda x: x.value)

        # HUMAN_REVIEW: high-value + borderline + short employment
        high_value = app.get('loan_amount', 0) > 1_000_000_000
        low_conf   = decision.get('confidence', 1.0) < 0.65
        if high_value and low_conf:
            triggers.append(f"High-value loan >1B VNĐ + low confidence {decision['confidence']:.2f}")
            max_level = max(max_level, EscalationLevel.HUMAN_REVIEW, key=lambda x: x.value)

        # ALERT: fairness warning
        if 0.75 <= fairness_metrics.get('disparity_ratio', 1.0) < 0.80:
            triggers.append('Approaching 80% rule boundary')
            max_level = max(max_level, EscalationLevel.ALERT, key=lambda x: x.value)

        action_map = {
            EscalationLevel.NONE:         ("No action required", 0, "AI System"),
            EscalationLevel.LOG_ONLY:     ("Log and monitor", 0, "AI System"),
            EscalationLevel.ALERT:        ("Send alert to AI Team", 24, "AI Product Manager"),
            EscalationLevel.HUMAN_REVIEW: ("Manual review by Credit Officer", 48, "Credit Officer"),
            EscalationLevel.ETHICS_BOARD: ("Ethics Board review session", 72, "Ethics Board Chair"),
            EscalationLevel.HALT:         ("HALT AI decisions in affected segment", 1, "CRO + CEO"),
        }
        action, timeline, escalate_to = action_map[max_level]
        return EscalationDecision(max_level, triggers, action, timeline, escalate_to)

Khi ethics incident xảy ra (ví dụ: báo chí phát hiện bias), follow 4-phase playbook:

  1. Contain (giờ 0–4): Halt AI decisions trong affected segment. Assign incident commander (CRO). Document current state — tất cả evidence.
  2. Investigate (giờ 4–48): Root cause analysis — bias từ data, model, hay threshold? Quantify scope: bao nhiêu khách hàng bị ảnh hưởng? Từ khi nào?
  3. Remediate (giờ 48–168): Manual review cho affected cases. Fix root cause (update thresholds, retrain model, change data pipeline). Notify affected customers với explanation và reconsideration offer.
  4. Prevent (sau 168h): Post-mortem report. Update governance charter. Add new monitoring metric. Ethics Board review. Regulatory disclosure nếu required.

Thuật ngữ Định nghĩa
AI GovernanceFramework of policies, processes, và accountability structures đảm bảo AI systems hoạt động safely, ethically, và in compliance with law.
RACI MatrixResponsible-Accountable-Consulted-Informed. Clarifies who does what in AI decision chain. A (Accountable) chịu trách nhiệm pháp lý khi có sự cố.
EU AI ActEU regulation effective 2024 — classify AI systems theo risk (Minimal/Limited/High/Unacceptable). Credit scoring = High-Risk, cần documentation đầy đủ.
Demographic ParityFairness metric: approval rate phải bằng nhau giữa các demographic groups. P(approve|group A) ≈ P(approve|group B).
Equalized OddsFairness metric: TPR và FPR phải bằng nhau giữa groups. Stronger than demographic parity — focuses on accuracy equity.
80% Rule (Disparate Impact)Approval rate của protected group phải ≥ 80% của reference group. Legal standard from Equal Credit Opportunity Act và EU AI Act.
Proxy DiscriminationModel dùng feature không phải protected attribute nhưng correlates strongly với nó — dẫn đến discriminatory outcomes dù không intentional.
Fairness ImpossibilityTheorem: không thể simultaneously satisfy Demographic Parity + Equalized Odds + Calibration khi base rates khác nhau giữa groups.
XAI (Explainable AI)Methods để make AI decisions interpretable: LIME (local surrogate), SHAP (feature attribution), counterfactual explanations, natural language explanations.
Counterfactual Explanation"Minimum change" explanation: thay đổi gì ít nhất để quyết định flip. "Nếu employment ≥ 12mo → APPROVE." Actionable cho customer.
Right to ExplanationGDPR Art. 22 + EU AI Act Art. 86: người dùng có quyền yêu cầu meaningful explanation cho automated decisions affecting them.
Model CardStandardized documentation của AI model: intended use, performance metrics, limitations, fairness metrics, và versioning history.
Ethics Review BoardCross-functional body (Legal, Risk, Data Science, External Advisor) reviewing AI ethics concerns. Advisory power — khuyến nghị, không veto.
Escalation PathDefined chain: AI System → AI Engineer → Risk Manager → CRO → NHNN. Severity determines at which level decision must be made.
Compliance Evidence PackageTập hợp documents chứng minh compliance với regulations. Bao gồm: decision logs, bias metrics, explainability records, audit trails — generated tự động.

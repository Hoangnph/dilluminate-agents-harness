1.1 Agentic Loop: Model ↔ Tools

Hình 1: Tool Calling Agentic Loop — Claude phát ra tool_use → Harness execute → tool_result trả về → Claude tiếp tục reasoning cho đến khi stop_reason="end_turn"

import anthropic

client = anthropic.Anthropic()

# Định nghĩa tool schema cho CreditBureauTool
CREDIT_BUREAU_TOOL = {
    "name":        "get_credit_score",
    "description": "Lấy điểm tín dụng và lịch sử từ CIC NHNN cho khách hàng.",
    "input_schema": {
        "type": "object",
        "properties": {
            "customer_id": {
                "type":        "string",
                "description": "Mã khách hàng FinTech Corp (format: FC-YYYY-NNN)"
            },
            "id_number": {
                "type":        "string",
                "description": "CCCD/CMND — 12 chữ số"
            },
            "include_history": {
                "type":        "boolean",
                "description": "True để lấy 24 tháng payment history",
                "default":     True
            }
        },
        "required": ["customer_id", "id_number"]
    }
}

def run_loanbot_agent(loan_application: dict) -> dict:
    # Agentic loop: chạy cho đến khi stop_reason = "end_turn"
    messages = [{
        "role": "user",
        "content": f"Thẩm định hồ sơ vay:\n{loan_application}"
    }]

    while True:
        response = client.messages.create(
            model="claude-haiku-4-5-20251001",
            max_tokens=4096,
            tools=[CREDIT_BUREAU_TOOL, COMPLIANCE_TOOL, INCOME_VERIFICATION_TOOL],
            messages=messages
        )
        # Nếu model dùng tools → execute và loop lại
        if response.stop_reason == "tool_use":
            tool_results = execute_tools(response.content)
            messages.append({"role": "assistant", "content": response.content})
            messages.append({"role": "user",      "content": tool_results})
        elif response.stop_reason == "end_turn":
            return {"assessment": response.content[0].text}
        else:
            raise RuntimeError(f"Unexpected stop: {response.stop_reason}")
🧠 Kiểm tra nhanh

Trong agentic loop, khi nào thì dừng (exit loop)? Điều gì xảy ra nếu model tiếp tục gọi tools vô hạn?

Gợi ý: Loop dừng khi stop_reason == "end_turn" — model không còn tool_use blocks, chỉ có text response. Nếu model gọi tools vô hạn (infinite loop): phải có max_iterations guard trong harness — e.g., nếu vòng lặp > 10 → raise error. Trong production LoanBot, max_iterations=5 đủ (CreditBureau + Compliance + Income = 3 tools, có thể retry = max 5). Ngoài ra, total token budget giới hạn naturally ngăn infinite loop.

Bài 2: Enterprise Tool Design — Idempotency, Caching, Rate Limiting

🎯 Sau bài này, bạn sẽ:

  • Thiết kế tool functions an toàn cho production: idempotent, cacheable, rate-limited
  • Implement tool-level caching với TTL phù hợp từng loại data
  • Xây dựng rate limiter cho external API calls
  • Phân biệt read-only vs write tools và implications
💡 Analogy: Tools như công tắc điện

Read-only tool (CreditBureauTool) = công tắc đèn bình thường — bật tắt nhiều lần cũng không sao. Write tool (NotificationTool) = chuông cửa — gọi hai lần sẽ rung hai lần → customer nhận 2 SMS. Idempotency = đảm bảo gọi nhiều lần = gọi một lần. Với write tools: hash(params) làm idempotency key, lưu trong Redis. Nếu đã gửi → skip silently.

Hình 3: ToolRegistry Architecture — 3 Production Guards bảo vệ external API calls: Cache (giảm load), Rate Limiter (tránh quota exceeded), Idempotency (mỗi write chỉ chạy 1 lần)
Python 
import hashlib, time, functools
from dataclasses import dataclass
from typing import Dict, Optional, Any
from collections import deque

@dataclass
class ToolConfig:
    name:          str
    ttl_seconds:   int   = 0     # 0 = no cache
    rate_limit_rps:int   = 10    # requests per second
    timeout_s:     float = 5.0
    is_write:      bool  = False  # write tools need idempotency key

# Tool configs cho LoanBot production
TOOL_CONFIGS = {
    "credit_bureau":      ToolConfig("credit_bureau",       ttl_seconds=86_400, rate_limit_rps=5),
    "property_valuation": ToolConfig("property_valuation",  ttl_seconds=604_800, rate_limit_rps=2),
    "compliance_check":   ToolConfig("compliance_check",    ttl_seconds=0,       rate_limit_rps=20),
    "income_verification":ToolConfig("income_verification", ttl_seconds=2_592_000, rate_limit_rps=3),
    "send_notification":  ToolConfig("send_notification",   ttl_seconds=0,       rate_limit_rps=50, is_write=True),
}

class TokenBucketRateLimiter:
    """Token bucket: allows burst up to capacity, refills at rate_rps/s."""
    def __init__(self, rate_rps: int, capacity: int = 10):
        self.rate      = rate_rps
        self.capacity  = capacity
        self.tokens    = float(capacity)
        self.last_tick = time.monotonic()

    def acquire(self) -> bool:
        now    = time.monotonic()
        delta  = now - self.last_tick
        self.tokens    = min(self.capacity, self.tokens + delta * self.rate)
        self.last_tick = now
        if self.tokens >= 1:
            self.tokens -= 1
            return True
        return False  # caller should backoff or queue

class ToolRegistry:
    """Central registry cho tất cả LoanBot tools với caching + rate limiting."""

    def __init__(self):
        self._cache:   Dict[str, tuple] = {}           # key → (value, expire_time)
        self._limiters = {                              # tool_name → limiter
            name: TokenBucketRateLimiter(cfg.rate_limit_rps)
            for name, cfg in TOOL_CONFIGS.items()
        }
        self._idempotency_set: set = set()

    def call(self, tool_name: str, params: Dict, impl) -> Any:
        cfg = TOOL_CONFIGS[tool_name]
        # Idempotency check for write tools
        if cfg.is_write:
            idem_key = hashlib.md5(f"{tool_name}:{params}".encode()).hexdigest()
            if idem_key in self._idempotency_set:
                return {"status": "skipped", "reason": "already_sent"}
            self._idempotency_set.add(idem_key)
        # Cache check for read tools
        cache_key = f"{tool_name}:{sorted(params.items())}"
        if cfg.ttl_seconds > 0:
            cached = self._get_cache(cache_key)
            if cached: return cached
        # Rate limit
        if not self._limiters[tool_name].acquire():
            raise RuntimeError(f"{tool_name}: rate limit exceeded")
        result = impl(**params)
        if cfg.ttl_seconds > 0:
            self._set_cache(cache_key, result, cfg.ttl_seconds)
        return result

    def _get_cache(self, key: str) -> Optional[Any]:
        if key in self._cache:
            val, exp = self._cache[key]
            if time.time() < exp: return val
        return None

    def _set_cache(self, key: str, val: Any, ttl: int):
        self._cache[key] = (val, time.time() + ttl)

Bài 3: Parallel Tool Orchestration — Khi nào gọi đồng thời

🎯 Sau bài này, bạn sẽ:

  • Nhận biết khi Claude phát ra multiple tool_use blocks cùng lúc
  • Execute parallel tool calls với asyncio.gather
  • Handle mixed success/failure trong parallel tool batch
  • Đo speedup: FC-2024-004 từ 4.4s → 2.1s với parallel tools

Claude có thể phát ra nhiều tool_use blocks trong cùng một response khi nhận thấy các tools độc lập nhau. Harness phải nhận ra pattern này và execute chúng song song thay vì tuần tự.

📌 Khái niệm: Parallel Tool Blocks

Khi Claude response chứa nhiều type: "tool_use" blocks, đây là signal để execute song song. Ví dụ: Claude cần cả credit score lẫn compliance check để assess FC-2024-004 — hai thông tin này độc lập nhau. Sequential: 1.2s + 0.6s = 1.8s. Parallel: max(1.2, 0.6) = 1.2s. Harness phải detect multiple tool_use → asyncio.gather → collect all results → trả về dưới dạng list of tool_result blocks.

Hình 4: asyncio.gather() Parallel Execution — 4 tools chạy đồng thời, tổng thời gian = tool chậm nhất (PropertyValuation 2.1s). Speedup 61% so với sequential (5.4s)

import asyncio

async def execute_tools_parallel(tool_uses: list, registry: ToolRegistry) -> list:
    """Execute tất cả tool_use blocks đồng thời, collect results."""
    async def _run_one(tool_use_block) -> dict:
        tool_name = tool_use_block.name
        params    = tool_use_block.input
        try:
            # registry.call is sync — wrap in executor for async context
            result = await asyncio.get_event_loop().run_in_executor(
                None,
                lambda: registry.call(tool_name, params, TOOL_IMPLS[tool_name])
            )
            return {
                "type":       "tool_result",
                "tool_use_id": tool_use_block.id,
                "content":     str(result)
            }
        except Exception as e:
            return {
                "type":       "tool_result",
                "tool_use_id": tool_use_block.id,
                "content":     f"ERROR: {e}",
                "is_error":    True
            }

    # gather tất cả — return_exceptions để không chết khi 1 tool fail
    tasks   = [_run_one(tu) for tu in tool_uses]
    results = await asyncio.gather(*tasks, return_exceptions=True)
    return [r for r in results if not isinstance(r, Exception)]

Bài 4: Schema Design & Validation — Pydantic + JSON Schema

🎯 Sau bài này, bạn sẽ:

  • Thiết kế strict JSON Schema cho LoanBot tool inputs/outputs
  • Validate tool outputs với Pydantic trước khi pass cho Claude
  • Xử lý schema evolution — thêm fields mà không break cũ
  • Thiết kế structured outputs với claude API (JSON mode)

Tool schemas không chỉ là documentation — chúng là validation contracts. Nếu Claude gọi get_credit_score với customer_id: 12345 (integer) thay vì "FC-2024-004" (string) → tool sẽ fail. Schema phải catch sai lầm này trước khi reach production API.

⚠️ Cảnh báo: Schema quá lỏng → Hallucination tăng

Tool schema với chỉ "type": "string" cho mọi field → Claude có thể pass bất cứ thứ gì. Ví dụ: id_number: "twelve digits" thay vì 12 chữ số thực tế. Strict schema với pattern: "^\\d{12}$" + description rõ ràng → Claude generate đúng format rate 97% vs 78% khi schema lỏng (LoanBot A/B test). Invest time vào schema design upfront.

Hình 5: Schema Validation Pipeline — Raw input phải qua JSON Schema + Pydantic + Business Rules trước khi gọi external API. Schema chặt → Claude generate đúng format rate 97% vs 78%

from pydantic import BaseModel, Field, validator
from typing import Literal, Optional
import re

class CreditBureauInput(BaseModel):
    """Validated input cho CreditBureauTool."""
    customer_id:     str = Field(pattern=r"^FC-\d{4}-\d{3,}$",
                                description="Format: FC-YYYY-NNN")
    id_number:       str = Field(pattern=r"^\d{12}$",
                                description="CCCD 12 chữ số")
    include_history: bool = True

class CreditBureauOutput(BaseModel):
    """Validated output từ CIC — phải validate trước khi trả cho Claude."""
    credit_score:      int           = Field(ge=300, le=850)
    credit_grade:      Literal['A','B','C','D']
    payment_history:   str           = Field(description="24-month payment pattern")
    derogatory_marks:  int           = Field(ge=0)
    active_loans:      int           = Field(ge=0)
    retrieved_at:      str           # ISO timestamp

class LoanDecisionOutput(BaseModel):
    """Structured output từ RiskAgent — JSON mode để parse chắc chắn."""
    loan_id:           str
    decision:          Literal['APPROVE', 'REJECT', 'REVIEW', 'HITL']
    risk_score:        float         = Field(ge=0.0, le=1.0)
    confidence:        float         = Field(ge=0.0, le=1.0)
    key_factors:       list[str]
    regulation_refs:   list[str]     # e.g. ["TT39 Điều 15", "CV5627 Khoản 3"]
    hitl_required:     bool
    hitl_level:        Optional[int]  = Field(None, ge=1, le=3)

def parse_tool_input(tool_name: str, raw_input: dict) -> BaseModel:
    """Validate Claude's tool input trước khi call external API."""
    VALIDATORS = {
        "get_credit_score": CreditBureauInput,
    }
    validator_cls = VALIDATORS.get(tool_name)
    if not validator_cls:
        raise ValueError(f"Unknown tool: {tool_name}")
    return validator_cls(**raw_input)  # raises ValidationError if invalid
🧠 Kiểm tra nhanh

Tại sao LoanDecisionOutput cần regulation_refs: list[str]? Không thể chỉ cần decision: str sao?

Gợi ý: EU AI Act Art.13 yêu cầu automated creditworthiness decisions phải explainable và auditable. regulation_refs = list các điều khoản pháp lý được áp dụng (e.g., "TT39 Điều 15") → loan officer có thể verify từng claim. Chỉ có decision: str ("APPROVE") không đủ để audit trail. Đây cũng là business requirement: nếu customer khiếu nại → phải cite cụ thể regulation nào dẫn đến rejection.

Bài 5: Error Recovery & Tool Resilience

🎯 Sau bài này, bạn sẽ:

  • Thiết kế graceful error messages cho failed tools
  • Implement exponential backoff retry cho transient failures
  • Xây dựng Circuit Breaker để ngăn cascade failures
  • Define fallback behavior khi tool critical fail

Trong production, external tools fail — CIC timeout, PropertyValuation API down, compliance service overloaded. LoanBot cần xử lý những failures này gracefully thay vì crash toàn bộ pipeline.

🔑 Key Insight: Tool Failure ≠ Pipeline Failure

Không phải mọi tool failure đều phải kill loan assessment. CreditBureau timeout → có thể retry với cache, hoặc dùng cached score (stale nhưng acceptabel cho initial assessment). PropertyValuation unavailable → fallback: "Cần thẩm định thủ công" + flag for HITL. ComplianceCheck fail → đây là critical: KHÔNG thể approve, phải HITL. Classify mỗi tool: CRITICAL (failure = block), DEGRADED (failure = fallback), OPTIONAL (failure = skip).

Hình 2: Circuit Breaker States — CLOSED (normal) → OPEN (block calls after 5 failures) → HALF-OPEN (test) → CLOSED (recover) hoặc OPEN lại
Python 
from enum import Enum
import time

class CBState(Enum):
    CLOSED    = 'CLOSED'
    OPEN      = 'OPEN'
    HALF_OPEN = 'HALF_OPEN'

class CircuitBreaker:
    """Circuit breaker cho LoanBot tools — ngăn cascade failures."""

    def __init__(self, failure_threshold: int = 5,
                       cooldown_s: float = 60.0):
        self.failure_threshold = failure_threshold
        self.cooldown_s        = cooldown_s
        self.state             = CBState.CLOSED
        self.failure_count     = 0
        self.opened_at: Optional[float] = None

    def call(self, fn, *args, **kwargs):
        if self.state == CBState.OPEN:
            # Check if cooldown expired
            if time.time() - self.opened_at >= self.cooldown_s:
                self.state = CBState.HALF_OPEN
            else:
                raise RuntimeError("Circuit breaker OPEN — service unavailable")
        try:
            result = fn(*args, **kwargs)
            if self.state == CBState.HALF_OPEN:
                self._reset()  # trial succeeded → fully CLOSED
            return result
        except Exception as e:
            self._on_failure()
            raise

    def _on_failure(self):
        self.failure_count += 1
        if self.failure_count >= self.failure_threshold or self.state == CBState.HALF_OPEN:
            self.state     = CBState.OPEN
            self.opened_at = time.time()

    def _reset(self):
        self.state = CBState.CLOSED
        self.failure_count = 0
        self.opened_at = None

💰 Agent Economics & ROI Analysis

Sau 28 module xây dựng LoanBot, câu hỏi cuối cùng với Board: "Đáng tiền không?" Module này builds ROI framework từ nguyên liệu thô — token costs, staff savings, accuracy improvements, và governance overhead — thành một business case thuyết phục với NPV, payback period, và sensitivity analysis mà CFO có thể ký duyệt.

Bài 1: ROI Framework — Từ Chi Phí đến Giá Trị

🎯 Sau bài này, bạn sẽ:

  • Xác định tất cả cost components và benefit components của LoanBot deployment
  • Tính Total Cost of Ownership (TCO): development + API + infra + governance + staff
  • Tính Net Business Value: trực tiếp (staff savings, speed) và gián tiếp (quality, compliance)
  • Build ROI formula và interpret kết quả cho C-Suite audience
🏢 Tình huống: FinTech Corp trước và sau LoanBot

Trước LoanBot (Manual): 200 nhân viên thẩm định, mỗi hồ sơ tốn 350.000 VNĐ chi phí trực tiếp (lương + overhead), xử lý 50.000 hồ sơ/tháng = 17.5 tỷ VNĐ/tháng operating cost. Processing time: 3–5 ngày. Error rate (human): ~12% Type I errors → ~6.000 bad loans/tháng. Sau LoanBot v4.0: 25 nhân viên (REVIEW cases + oversight), API cost + infra + governance overhead, xử lý 50.000 hồ sơ/tháng = 5.9 tỷ VNĐ/tháng. Processing time: 5 phút. Error rate: 8.2% Type I → ~4.100 bad loans/tháng → tiết kiệm ~1.900 × 180M = 342 tỷ/tháng bad loan reduction. Tuy nhiên: 342 tỷ là incremental savings từ accuracy improvement (M27), không phải operational savings thuần túy.

ROI Formula
ROI = (Net Benefit / Total Investment) × 100%
Net Benefit = Total Savings − Total Operating Cost of AI. Total Savings = Staff Cost Reduction + Error Cost Reduction + Speed Value + Compliance Value. Total Investment = Development Cost + Infrastructure Setup. Tất cả tính trên cùng time horizon (thường 3 năm).

🧠 Kiểm tra nhanh

FinTech Corp đầu tư 2.3 tỷ VNĐ để build LoanBot. Net savings là 12.677 tỷ/tháng. Payback period là bao nhiêu ngày? Và ROI 3-năm?

Gợi ý: Payback period = 2.3 tỷ / 12.677 tỷ × 30 ngày = 5.5 ngày. ROI 3-năm: Net Benefit = 12.677 tỷ × 36 tháng − 2.3 tỷ = 453.4 tỷ. ROI = 453.4 / 2.3 × 100% = 19.713%. Đây là ROI extraordinary — trong thực tế, con số này trigger câu hỏi "Tại sao chưa làm sớm hơn?" từ Board. Payback period 5.5 ngày là lý do mọi financial services công ty đều đang race để deploy AI.

Bài 2: Token Economics — Chi Phí Thực Tế của Mỗi Quyết Định

🎯 Sau bài này, bạn sẽ:

  • Tính chính xác cost-per-decision dựa trên token consumption của từng agent call
  • Phân tích tradeoff: simple routing agent (Haiku) vs multi-agent debate (Sonnet/Opus)
  • Implement token budget cho từng decision tier để cap maximum cost/case
  • Tối ưu prompt length để giảm input token cost mà không giảm quality
💡 Analogy: Token Cost như Điện thoại theo Phút

Thời kỳ điện thoại tính cước theo phút: gọi ngắn là rẻ, gọi dài tốn tiền. LLM API tính theo token: prompt ngắn + output ngắn = rẻ. Khác biệt quan trọng: bạn trả tiền cho cả input lẫn output — và input của mỗi agent call thường mang toàn bộ conversation history (= cumulative cost problem trong multi-turn). FinTech Corp cần biết: "Mỗi loan decision tốn bao nhiêu token — và tôi có thể predict cost không?" Câu trả lời là YES — nếu bạn thiết kế system với token budgets rõ ràng.

Hình 1: Token cost breakdown cho 3 tiers. Tier 1 (80% volume) rất rẻ nhờ Haiku + short prompts. Tier 3 (3% volume, high-value loans >2B VNĐ) dùng 5-agent panel với Sonnet — tốn hơn nhưng justified bởi loan size. Weighted average: ~115 VNĐ/decision.
Python (Token Budget Manager) 
from dataclasses import dataclass
from typing import Dict, Optional
from enum import Enum

class DecisionTier(Enum):
    CLEAR     = "CLEAR"      # 80% — Haiku single pass
    STANDARD  = "STANDARD"   # 17% — 3-agent Sonnet debate
    HIGH_VALUE = "HIGH_VALUE" # 3%  — 5-agent panel

@dataclass
class TokenBudget:
    tier: DecisionTier
    max_input_tokens: int
    max_output_tokens: int
    model: str
    n_agents: int
    max_cost_usd: float       # Hard cap per decision
    max_cost_vnd: int

# Token budgets per tier — enforced at runtime
TIER_BUDGETS: Dict[DecisionTier, TokenBudget] = {
    DecisionTier.CLEAR: TokenBudget(
        tier=DecisionTier.CLEAR,
        max_input_tokens=1500, max_output_tokens=400,
        model="claude-haiku-4-5-20251001", n_agents=1,
        max_cost_usd=0.005, max_cost_vnd=125
    ),
    DecisionTier.STANDARD: TokenBudget(
        tier=DecisionTier.STANDARD,
        max_input_tokens=3500, max_output_tokens=1000,
        model="claude-sonnet-4-6", n_agents=3,
        max_cost_usd=0.025, max_cost_vnd=625
    ),
    DecisionTier.HIGH_VALUE: TokenBudget(
        tier=DecisionTier.HIGH_VALUE,
        max_input_tokens=4500, max_output_tokens=1200,
        model="claude-sonnet-4-6", n_agents=5,
        max_cost_usd=0.060, max_cost_vnd=1500
    ),
}

class TokenBudgetManager:
    USD_TO_VND = 25_000
    # claude-haiku-4-5: $0.80/MTok in, $4/MTok out
    # claude-sonnet-4-6: $3/MTok in, $15/MTok out
    PRICING = {
        "claude-haiku-4-5-20251001": (0.80,  4.00),
        "claude-sonnet-4-6":         (3.00, 15.00),
    }

    def classify_tier(self, loan_amount: int, credit_score: int,
                        dti: float) -> DecisionTier:
        # Clear REJECT or APPROVE — no debate needed
        if credit_score < 500 or dti > 0.55: return DecisionTier.CLEAR
        if credit_score >= 720 and dti <= 0.35: return DecisionTier.CLEAR
        # High-value borderline — 5-agent panel
        if loan_amount > 2_000_000_000 and 580 <= credit_score <= 700:
            return DecisionTier.HIGH_VALUE
        return DecisionTier.STANDARD

    def estimate_cost(self, tier: DecisionTier,
                        actual_tokens: Optional[Dict] = None) -> dict:
        """Estimate or compute actual cost for a decision."""
        budget = TIER_BUDGETS[tier]
        tokens = actual_tokens or {
            'input': budget.max_input_tokens * budget.n_agents * 0.85,
            'output': budget.max_output_tokens * budget.n_agents * 0.7
        }
        price_in, price_out = self.PRICING[budget.model]
        cost_usd = (tokens['input'] * price_in + tokens['output'] * price_out) / 1_000_000
        cost_vnd = int(cost_usd * self.USD_TO_VND)
        over_budget = cost_usd > budget.max_cost_usd
        return {'tier': tier.value, 'cost_usd': cost_usd, 'cost_vnd': cost_vnd,
                'over_budget': over_budget, 'budget_usd': budget.max_cost_usd}

# Monthly cost estimate — 50K cases/month
mgr = TokenBudgetManager()
monthly_cases = {DecisionTier.CLEAR: 40000, DecisionTier.STANDARD: 8500, DecisionTier.HIGH_VALUE: 1500}
total_vnd = sum(mgr.estimate_cost(tier)['cost_vnd'] * count for tier, count in monthly_cases.items())
print(f"Monthly API cost estimate: {total_vnd:,} VNĐ ({total_vnd/50000:.0f} VNĐ/case avg)")

Bài 3: Scaling Economics — Chi Phí Biên và Điểm Hòa Vốn

🎯 Sau bài này, bạn sẽ:

  • Hiểu fixed vs variable costs trong AI deployment: tại sao marginal cost gần bằng 0
  • Tính Break-Even Point: khi nào LoanBot rẻ hơn hiring thêm nhân viên
  • Model cost curves: LoanBot scale tuyến tính (linear) còn human team scale step-wise
  • Analyze economies of scale: tại sao unit cost giảm khi volume tăng (fixed cost amortization)
🔑 Key Insight: AI Có Marginal Cost Gần Bằng 0

Thêm 1 nhân viên thẩm định: +175M VNĐ/tháng cố định (lương + overhead). Thêm 1.000 hồ sơ/tháng cho AI: +115.000 VNĐ (1.000 × 115 VNĐ/case). Đây là sự khác biệt căn bản giữa human labor và AI: AI scale tuyến tính với marginal cost thấp, trong khi human scale step-wise với marginal cost cao. Ý nghĩa chiến lược: FinTech Corp có thể expand sang rural markets (volume tăng 30%) với 0 additional headcount — chỉ thêm API calls. Human team cần thêm 60 nhân viên cho 30% volume increase.

Hình 2: Cost scaling comparison. Human team (đỏ): tăng step-wise mỗi khi cần hire thêm batch nhân viên. LoanBot (xanh): tăng tuyến tính với marginal cost 115 VNĐ/case. Break-even tại ~6.250 hồ sơ/tháng. FinTech Corp xử lý 50.000 — well above break-even.
🧠 Kiểm tra nhanh

FinTech Corp muốn expand sang rural markets — dự kiến volume tăng thêm 15.000 hồ sơ/tháng (+30%). Với human team, chi phí tăng bao nhiêu? Với LoanBot?

Gợi ý: Human team: cần thêm 15.000/250 hồ sơ/người × 175M/người = 60 người × 175M = 10.5 tỷ/tháng tăng thêm. LoanBot: 15.000 × 115 VNĐ = 1.725.000 VNĐ ≈ 1.73 triệu/tháng tăng thêm. Ratio: Human expensive by 10.5B/1.73M ≈ 6.070 lần. Đây là lý do core business case: AI allows "elastic scaling" — FinTech Corp có thể serve rural markets (mission + revenue) với marginal cost gần bằng 0, điều không thể về mặt kinh tế với human team.

Bài 4: Business Case Construction — Từ Số Liệu đến Quyết Định

🎯 Sau bài này, bạn sẽ:

  • Build 3-year financial model: NPV, IRR, Payback Period cho LoanBot investment
  • Identify và quantify intangible benefits: speed-to-market, compliance, brand reputation
  • Perform sensitivity analysis: điều gì xảy ra nếu assumptions sai?
  • Structure executive presentation: CFO muốn thấy gì trong 5 slides đầu tiên
⚠️ Sai Lầm Phổ Biến: Chỉ Count Direct Cost Savings

Nhiều teams chỉ present "tiết kiệm nhân sự" và bỏ qua 4 loại value khác: (1) Speed Value: 5 phút vs 3 ngày → khách hàng chấp nhận mức lãi suất thấp hơn 0.5% vì convenience → revenue impact lớn. (2) Compliance Value: Tránh NHNN penalty 500M VNĐ/incident × probability. (3) Accuracy Value: Type I error reduction → bad loan reduction → capital preservation. (4) Market Expansion Value: Rural markets previously infeasible → new revenue stream. Khi include tất cả 4 loại, ROI thường 3–5× higher than direct savings alone.

Python (3-Year NPV Calculator) 
from dataclasses import dataclass, field
from typing import List
import math

@dataclass
class YearlyCashFlow:
    year: int
    revenue_savings: int          # Staff cost reduction
    error_savings: int           # Bad loan reduction value
    speed_value: int             # Revenue from speed premium
    compliance_value: int        # Avoided penalty exposure
    api_costs: int              # LLM API bill
    infra_costs: int            # Cloud + monitoring
    governance_costs: int       # Ethics board, audits
    capex: int                  # Year 0: development investment

    @property
    def net_cash_flow(self) -> int:
        benefits = self.revenue_savings + self.error_savings + self.speed_value + self.compliance_value
        costs    = self.api_costs + self.infra_costs + self.governance_costs + self.capex
        return benefits - costs

class BusinessCaseCalculator:
    DISCOUNT_RATE = 0.12   # 12% — typical enterprise WACC for Vietnam fintech
    VND_B = 1_000_000_000

    def build_3year_model(self) -> List[YearlyCashFlow]:
        # Month → Year scaling (12x, with growth assumptions)
        return [
            # Year 0: Development + Pilot (6 months actual operation)
            YearlyCashFlow(0,
                revenue_savings=13_125_000_000 * 6,    # 6mo partial operation
                error_savings=342_000_000_000 * 6,
                speed_value=2_000_000_000 * 6,
                compliance_value=800_000_000 * 6,
                api_costs=340_000_000 * 6,
                infra_costs=500_000_000 * 6,
                governance_costs=150_000_000 * 6,
                capex=2_300_000_000   # Development + setup
            ),
            # Year 1: Full operation + 10% volume growth
            YearlyCashFlow(1,
                revenue_savings=13_125_000_000 * 12,
                error_savings=342_000_000_000 * 12,
                speed_value=2_000_000_000 * 12,
                compliance_value=800_000_000 * 12,
                api_costs=340_000_000 * 12,
                infra_costs=500_000_000 * 12,
                governance_costs=150_000_000 * 12,
                capex=0
            ),
            # Year 2: +20% volume (rural expansion)
            YearlyCashFlow(2,
                revenue_savings=13_125_000_000 * 14,   # Slightly more staff savings
                error_savings=342_000_000_000 * 14,
                speed_value=2_000_000_000 * 14,
                compliance_value=800_000_000 * 14,
                api_costs=340_000_000 * 14,
                infra_costs=600_000_000 * 14,
                governance_costs=200_000_000 * 14,
                capex=0
            ),
        ]

    def npv(self, cashflows: List[YearlyCashFlow]) -> int:
        return int(sum(
            cf.net_cash_flow / ((1 + self.DISCOUNT_RATE) ** cf.year)
            for cf in cashflows
        ))

    def roi(self, cashflows: List[YearlyCashFlow]) -> float:
        total_benefit = sum(cf.net_cash_flow + cf.capex for cf in cashflows if cf.net_cash_flow > 0)
        total_invest  = sum(cf.capex for cf in cashflows)
        return (total_benefit / total_invest * 100) if total_invest > 0 else 0

Bài 5: Continuous Value Monitoring — Đảm Bảo ROI Thực Tế

🎯 Sau bài này, bạn sẽ:

  • Define Value KPIs: phân biệt leading indicators (hiện tại) vs lagging indicators (kết quả)
  • Build Value Dashboard: real-time tracking của cost-per-decision, accuracy, và revenue impact
  • Implement Value Attribution: khi có nhiều improvement đồng thời, attribute credit đúng chỗ
  • Detect Value Erosion: khi model drift dẫn đến ROI giảm — alert trước khi leadership notice
📌 Khái niệm: Leading vs Lagging Value Indicators

Lagging Indicators (kết quả đã xảy ra, chậm vài tháng): Default rate, Revenue from new loans, P&L impact. Vấn đề: bạn biết ROI đang giảm SAU KHI nó đã giảm 3 tháng. Leading Indicators (predict future outcomes, real-time): Decision confidence distribution, Type I error rate, Explanation quality score, Model PSI drift. Tốt: bạn biết ROI SẮP giảm trước khi nó xảy ra → can intervene. Rule: monitor leading indicators daily, review lagging indicators monthly, report both to Board quarterly.

Python (Value Dashboard — Real-time KPI Tracker) 
from dataclasses import dataclass, field
from typing import Dict, List
from datetime import datetime

@dataclass
class ValueKPI:
    name: str
    current: float
    target: float
    unit: str
    kpi_type: str      # 'leading' / 'lagging'
    direction: str    # 'lower_better' / 'higher_better'

    @property
    def on_track(self) -> bool:
        if self.direction == 'lower_better': return self.current <= self.target
        return self.current >= self.target

    @property
    def variance_pct(self) -> float:
        return (self.current - self.target) / self.target * 100

class ValueDashboard:
    def __init__(self):
        self.kpis: List[ValueKPI] = [
            # Leading indicators — monitored daily
            ValueKPI("Cost Per Decision", 118, 125, "VNĐ", "leading", "lower_better"),
            ValueKPI("Type I Error Rate", 0.082, 0.100, "%", "leading", "lower_better"),
            ValueKPI("Decision Confidence P50", 0.78, 0.75, "score", "leading", "higher_better"),
            ValueKPI("PSI Drift Score", 0.08, 0.10, "score", "leading", "lower_better"),
            ValueKPI("Fairness Disparity Ratio", 0.87, 0.80, "ratio", "leading", "higher_better"),
            # Lagging indicators — reviewed monthly
            ValueKPI("30-Day Default Rate", 0.042, 0.050, "%", "lagging", "lower_better"),
            ValueKPI("Monthly Net Savings", 12.4, 12.0, "tỷ VNĐ", "lagging", "higher_better"),
            ValueKPI("Customer NPS (Speed)", 72, 70, "score", "lagging", "higher_better"),
        ]

    def report(self) -> dict:
        """Generate value dashboard snapshot."""
        on_track = [k for k in self.kpis if k.on_track]
        off_track = [k for k in self.kpis if not k.on_track]
        alerts = [k for k in off_track if abs(k.variance_pct) > 15]
        return {'timestamp': datetime.now().isoformat(),
                'on_track': len(on_track), 'off_track': len(off_track),
                'alerts': [k.name for k in alerts],
                'overall_health': 'GREEN' if not alerts else ('YELLOW' if not off_track else 'RED')}

📚 Glossary Module 29

Thuật ngữ Định nghĩa
ROI (Return on Investment)(Net Benefit / Total Investment) × 100%. Percentage return trên mỗi đồng đầu tư. AI ROI thường 1.000%+ trong năm đầu do marginal cost thấp.
NPV (Net Present Value)Tổng cash flows tương lai chiết khấu về hiện tại. NPV > 0 → đầu tư sinh lời. Discount rate thường dùng WACC (12% cho Vietnam fintech).
TCO (Total Cost of Ownership)Tổng chi phí thực sự của AI deployment: development + API + infra + governance + training + maintenance. Không chỉ API cost.
Payback PeriodThời gian để thu hồi vốn đầu tư. LoanBot FinTech Corp: ~5.5 ngày. Thông thường enterprise projects: 12–24 tháng.
Cost-Per-Decision (CPD)Tổng AI operating cost / số decisions trong period. LoanBot v4.0: ~115 VNĐ/decision (weighted avg across 3 tiers). Key unit economics metric.
Token BudgetHard cap cho số token mỗi decision tier được phép dùng. Prevents runaway costs trong debate loops. Enforced at runtime — not just guideline.
Marginal CostChi phí để xử lý thêm 1 unit (1 hồ sơ). AI: ~115 VNĐ (variable). Human: ~175.000 VNĐ (loaded cost). AI marginal cost ≈ 1.500× thấp hơn human.
Break-Even PointVolume tại đó AI cost = Human cost. LoanBot: ~6.250 cases/month. Dưới ngưỡng này → human cheaper; trên ngưỡng → AI cheaper.
Fixed CostKhông thay đổi theo volume: infra (500M/tháng), governance overhead (150M). Amortized qua volume — unit cost giảm khi volume tăng.
Variable CostTỷ lệ với volume: API calls (340M/50K cases ≈ 6.800 VNĐ/case từ API thuần). Tăng linear khi volume tăng.
Leading KPIChỉ số predict future outcomes: PSI drift, Type I error rate, confidence distribution. Alert trước khi lagging indicators xấu.
Lagging KPIKết quả đã xảy ra: default rate, monthly savings, NPS. Accurate nhưng chậm (1–3 tháng lag) so với business events.
Value AttributionPhân bổ improvement credit đúng cho đúng thay đổi. Quan trọng khi nhiều updates deploy cùng thời điểm — tránh double-counting benefits.
Sensitivity AnalysisTest "what-if": nếu 1 assumption thay đổi ±X%, NPV thay đổi bao nhiêu? Xác định assumptions nào critical vs immaterial.
Value ErosionROI giảm dần theo thời gian do model drift, market changes, hoặc competitor actions. Early detection via leading KPIs.

MODULE 21 · Tuần 58–60

⚡ Real-time & Streaming Agents

LoanBot xử lý hồ sơ và nhân viên phải đợi 15–30 giây nhìn màn hình trắng. Streaming biến trải nghiệm đó thành text xuất hiện từng từ như AI đang "suy nghĩ thành tiếng" — TTFB giảm từ 18 giây xuống còn 0.3 giây.

⏱ 3 tuần 📖 5 bài giảng ⚡ Streaming APIs

Bài 1: Streaming Architecture — Tại Sao Real-time Matters?

🎯 Sau bài này, bạn sẽ:

  • Hiểu sự khác biệt giữa batch (chờ toàn bộ) và streaming (nhận từng phần)
  • Biết 4 protocols: HTTP polling, SSE, WebSocket, gRPC streaming
  • Chọn đúng protocol cho từng use case của LoanBot
  • Đo lường TTFB (Time to First Byte) và tại sao nó quan trọng hơn total latency
💡 Analogy: Nhà hàng vs FastFood

Batch processing = Nhà hàng truyền thống: bạn gọi món, đợi 20 phút, tất cả đồ ăn ra cùng lúc. Streaming = Conveyor belt sushi: từng món xuất hiện ngay khi sẵn sàng — bạn ăn được ngay trong khi những món khác vẫn đang được chuẩn bị. Với LoanBot: thay vì đợi toàn bộ phân tích xong mới hiện, nhân viên thấy kết quả xuất hiện dần — "Credit score: đạt ✓ ... DTI ratio: đang tính..."

Hình 1: Batch vs SSE Streaming cho LoanBot. Cùng tổng thời gian xử lý ~18 giây, nhưng TTFB khác nhau hoàn toàn: batch 18s (màn hình trắng) vs SSE 0.3s (text xuất hiện ngay). User experience cải thiện 60x dù backend không thay đổi.

4 Protocols: Chọn đúng cho đúng task

📡 HTTP Polling

Client hỏi server mỗi N giây: "xong chưa?"

✅ Simple, works everywhere
❌ Wasteful, N-second latency
📍 LoanBot: status check cũ (không dùng)

📻 Server-Sent Events

Server push text stream qua HTTP/1.1 (một chiều)

✅ Native browser support, auto-reconnect
❌ Unidirectional only
📍 LoanBot: loan analysis streaming ✅

🔌 WebSocket

Full-duplex connection — cả 2 chiều đồng thời

✅ Bidirectional, low latency
❌ Complex state management, proxy issues
📍 LoanBot: live Q&A với analyst ✅

🚀 gRPC Streaming

Binary protocol, Protobuf serialization, HTTP/2

✅ Efficient, typed, bidirectional
❌ No native browser support
📍 LoanBot: microservice-to-microservice ✅

Nguyên tắc chọn protocol
Browser UI → SSE · Service-to-service → gRPC · Interactive → WebSocket
SSE = simple unidirectional streaming to browser. WebSocket = bidirectional real-time. gRPC = efficient microservice communication.

Bài 2: Server-Sent Events — Streaming LoanBot Response

🎯 Sau bài này, bạn sẽ:

  • Hiểu SSE format: data: ...\n\n và các event types
  • Implement FastAPI streaming endpoint với StreamingResponse
  • Consume SSE từ browser với EventSource API
  • Handle errors và reconnection tự động
🏢 Tình huống: LoanBot SSE Dashboard

Nhân viên tín dụng mở hồ sơ FC-2024-002 (score 580, borderline). Thay vì đợi 15 giây, màn hình hiện ngay:

📊 Đang tải hồ sơ FC-2024-002...
✓ Credit score: 580 — dưới ngưỡng 620
✓ DTI ratio: 58% — vượt ngưỡng 45%
⏳ Đang phân tích lịch sử tín dụng
🔧 [Tool call: query_cic(FC-2024-002)]

Python / FastAPI (SSE Streaming Endpoint) 
from fastapi import FastAPI
from fastapi.responses import StreamingResponse
import anthropic, json, asyncio

app = FastAPI()
client = anthropic.AsyncAnthropic()

async def stream_loan_analysis(app_id: str, application: dict):
    """Generator yields SSE events as LoanBot streams analysis."""
    # Event 1: Start signal
    yield f'event: start\ndata: {{"app_id": "{app_id}"}}\n\n'

    prompt = f"""Analyze loan application {app_id}:
Credit Score: {application['credit_score']}
DTI Ratio: {application['dti_ratio']}
Employment: {application['employment_months']} months
Amount: {application['loan_amount']:,} VNĐ
Provide: step-by-step analysis, then final decision (APPROVE/REJECT/REVIEW)."""

    # Stream from Claude API
    async with client.messages.stream(
        model="claude-haiku-4-5-20251001",
        max_tokens=1024,
        messages=[{"role": "user", "content": prompt}],
    ) as stream:
        async for chunk in stream:
            if hasattr(chunk, 'delta') and hasattr(chunk.delta, 'text'):
                text = chunk.delta.text
                if text:
                    # SSE format: "data: \n\n"
                    payload = json.dumps({"type": "token", "text": text})
                    yield f'data: {payload}\n\n'

    # Event 3: Done signal with final parsed decision
    yield f'event: done\ndata: {{"status": "complete"}}\n\n'

@app.get("/stream/analyze/{app_id}")
async def analyze_stream(app_id: str):
    application = get_application(app_id)  # fetch from DB
    return StreamingResponse(
        stream_loan_analysis(app_id, application),
        media_type="text/event-stream",
        headers={
            "Cache-Control": "no-cache",
            "X-Accel-Buffering": "no",  # disable nginx buffering
            "Connection": "keep-alive",
        }
    )
JavaScript (Browser — EventSource consumer) 
// Browser: nhận SSE stream và cập nhật UI real-time
const streamLoanAnalysis = (appId, outputEl) => {
  const es = new EventSource(`/stream/analyze/${appId}`);

  es.addEventListener('start', (e) => {
    outputEl.innerHTML = '⏳ Đang phân tích...';
  });

  es.onmessage = (e) => {
    const data = JSON.parse(e.data);
    if (data.type === 'token') {
      outputEl.textContent += data.text;  // append incrementally
    }
  };

  es.addEventListener('done', (e) => {
    es.close();   // clean up connection
    outputEl.querySelector('.cursor')?.remove();
  });

  // Auto-reconnect on network error (built into EventSource)
  es.onerror = (e) => {
    if (es.readyState === EventSource.CLOSED) {
      console.log('Stream ended');
    }
    // EventSource auto-reconnects on transient errors
  };

  return es;  // return handle to cancel if needed
};
🧠 Kiểm tra nhanh

LoanBot đang stream analysis cho hồ sơ FC-2024-002. Nhân viên click "Hủy" sau 3 giây (đọc được đủ thông tin rồi). Cần làm gì ở cả client lẫn server để không lãng phí Claude API tokens?

Gợi ý: Client-side: gọi es.close() để đóng SSE connection → server nhận disconnect signal. Server-side: trong FastAPI generator, khi client disconnect, StreamingResponse sẽ raise GeneratorExit hoặc connection sẽ close. Quan trọng: phải wrap generator trong try/except để catch asyncio.CancelledError và cancel Claude stream: await stream.close(). Nếu không handle đúng, Claude API tiếp tục generate và charge tokens dù client không nhận nữa — lãng phí chi phí. Best practice: set max_tokens limit và implement server-side timeout (ví dụ: 30 giây max per stream).

Bài 3: WebSocket — Bidirectional Real-time Communication

🎯 Sau bài này, bạn sẽ:

  • Implement WebSocket endpoint với FastAPI
  • Cho phép nhân viên gửi follow-up questions trong khi LoanBot vẫn đang stream
  • Quản lý WebSocket connection lifecycle: connect, ping/pong, disconnect
  • Xây dựng connection pool cho nhiều analysts đồng thời
Hình 2: WebSocket session cho LoanBot Live Q&A. Analyst gửi questions bất kỳ lúc nào — server queue chúng và process tuần tự while streaming. Context window được maintain across toàn bộ conversation.
Python / FastAPI (WebSocket Manager) 
from fastapi import WebSocket, WebSocketDisconnect
import asyncio
from collections import defaultdict

class ConnectionManager:
    def __init__(self):
        self.active: dict[str, WebSocket] = {}
        self.queues: dict[str, asyncio.Queue] = defaultdict(asyncio.Queue)

    async def connect(self, ws: WebSocket, session_id: str):
        await ws.accept()
        self.active[session_id] = ws

    def disconnect(self, session_id: str):
        self.active.pop(session_id, None)
        self.queues.pop(session_id, None)

    async def send(self, session_id: str, msg: dict):
        ws = self.active.get(session_id)
        if ws:
            await ws.send_json(msg)

manager = ConnectionManager()

@app.websocket("/ws/{session_id}")
async def websocket_endpoint(ws: WebSocket, session_id: str):
    await manager.connect(ws, session_id)
    conversation_history = []  # maintain context across turns
    try:
        while True:
            question = await asyncio.wait_for(
                ws.receive_text(), timeout=90.0  # 90s idle timeout
            )
            conversation_history.append({"role": "user", "content": question})
            # Stream Claude response token by token
            async with client.messages.stream(
                model="claude-haiku-4-5-20251001", max_tokens=512,
                messages=conversation_history,
            ) as stream:
                full_response = ""
                async for chunk in stream:
                    if hasattr(chunk, 'delta') and hasattr(chunk.delta, 'text'):
                        if chunk.delta.text:
                            full_response += chunk.delta.text
                            await manager.send(session_id, {
                                "type": "token", "text": chunk.delta.text
                            })
            conversation_history.append({"role": "assistant", "content": full_response})
            await manager.send(session_id, {"type": "turn_done"})
    except (WebSocketDisconnect, asyncio.TimeoutError):
        manager.disconnect(session_id)

Bài 4: Streaming Tool Calls — Real-time Tool Execution

🎯 Sau bài này, bạn sẽ:

  • Hiểu tool call streaming: LLM stream text, dừng lại khi cần tool, stream tiếp
  • Xử lý input_json_delta events để reconstruct tool input
  • Hiển thị tool call progress real-time: "đang truy vấn CIC..."
  • Handle multiple parallel tool calls với asyncio.gather
📌 Khái niệm: Streaming Event Types

Claude API streaming trả về nhiều event types khác nhau. Với tool calls, cần handle đầy đủ:

  1. content_block_start — bắt đầu text block hoặc tool_use block
  2. content_block_delta với text_delta — token của response text
  3. content_block_delta với input_json_delta — partial JSON của tool input
  4. content_block_stop — kết thúc một block
  5. message_delta với stop_reason: "tool_use" — cần execute tool
  6. message_stop — toàn bộ response hoàn tất
Python (Streaming Tool Call Handler) 
import json, anthropic
from typing import AsyncGenerator

async def stream_with_tools(
    messages: list, tools: list
) -> AsyncGenerator[dict, None]:
    """Stream LoanBot response, handling tool calls transparently."""
    client = anthropic.AsyncAnthropic()

    while True:
        current_tool_calls = []
        current_tool_input_str = ""
        stop_reason = None
        response_content = []

        async with client.messages.stream(
            model="claude-sonnet-4-6",
            max_tokens=2048,
            tools=tools,
            messages=messages,
        ) as stream:
            async for event in stream.event_stream():
                if event.type == "content_block_start":
                    if event.content_block.type == "tool_use":
                        # Notify UI: tool call started
                        yield {"type": "tool_start",
                               "tool": event.content_block.name}
                        current_tool_calls.append({
                            "id":   event.content_block.id,
                            "name": event.content_block.name
                        })
                        current_tool_input_str = ""

                elif event.type == "content_block_delta":
                    if event.delta.type == "text_delta":
                        yield {"type": "text", "text": event.delta.text}
                    elif event.delta.type == "input_json_delta":
                        # Accumulate partial JSON for tool input
                        current_tool_input_str += event.delta.partial_json

                elif event.type == "message_delta":
                    stop_reason = event.delta.stop_reason

                elif event.type == "message_stop":
                    response_content = await stream.get_final_message()

        # Tool execution phase — after streaming ends
        if stop_reason == "tool_use" and current_tool_calls:
            tool_results = []
            for tc in current_tool_calls:
                tool_input = json.loads(current_tool_input_str)
                yield {"type": "tool_running", "tool": tc["name"], "input": tool_input}
                result = await execute_tool(tc["name"], tool_input)
                tool_results.append({
                    "type": "tool_result",
                    "tool_use_id": tc["id"],
                    "content": json.dumps(result)
                })
                yield {"type": "tool_done", "tool": tc["name"], "result": result}
            # Continue conversation with tool results
            messages = messages + [
                {"role": "assistant", "content": response_content.content},
                {"role": "user", "content": tool_results}
            ]
        else:
            break  # no more tool calls — done

Bài 5: Backpressure & Flow Control — Xử Lý Slow Consumers

🎯 Sau bài này, bạn sẽ:

  • Hiểu backpressure: khi consumer chậm hơn producer → buffer overflow
  • Implement queue-based buffering với asyncio.Queue
  • Thiết kế circuit breaker cho stream overload
  • Monitor streaming health: token rate, buffer depth, dropped connections
⚠️ Backpressure Problem trong LoanBot

FinTech Corp triển khai LoanBot streaming cho 200 nhân viên đồng thời. Claude API stream tokens nhanh (~50 tokens/giây), nhưng một số clients (mobile, slow connection) chỉ nhận được ~10 tokens/giây. Nếu không có backpressure: server buffer grow không giới hạn → OOM crash → tất cả 200 streams bị ngắt. Fix cần: queue với max size + slow client detection + graceful degradation.

Python (Backpressure với asyncio.Queue) 
import asyncio
from dataclasses import dataclass, field
from collections import deque
import time

@dataclass
class StreamHealth:
    bytes_sent:   int = 0
    tokens_sent:  int = 0
    buffer_depth: int = 0
    dropped:      bool = False
    start_time:   float = field(default_factory=time.time)

    def token_rate(self) -> float:
        elapsed = time.time() - self.start_time
        return self.tokens_sent / elapsed if elapsed > 0 else 0

async def buffered_stream(
    source_gen,       # async generator yielding tokens
    send_fn,          # coroutine to send to client
    max_buffer: int = 100,
    slow_threshold: float = 5.0   # tokens/sec — below = "slow client"
) -> StreamHealth:
    """Buffer LLM stream, detect slow clients, apply backpressure."""
    queue = asyncio.Queue(maxsize=max_buffer)
    health = StreamHealth()

    async def producer():
        """Read from LLM stream, put into queue. Block if full (backpressure)."""
        async for token in source_gen:
            try:
                await asyncio.wait_for(
                    queue.put(token),
                    timeout=2.0   # if queue full for 2s → slow client
                )
            except asyncio.TimeoutError:
                health.dropped = True
                break   # drop this slow client
        await queue.put(None)   # sentinel

    async def consumer():
        """Read from queue, send to client."""
        while True:
            token = await queue.get()
            health.buffer_depth = queue.qsize()
            if token is None: break
            await send_fn(token)
            health.tokens_sent += 1
            health.bytes_sent += len(token.encode())
            # Detect slow client and log for alerting
            if health.tokens_sent % 50 == 0 and health.token_rate() < slow_threshold:
                log_slow_client(health)

    # Run producer and consumer concurrently
    await asyncio.gather(producer(), consumer())
    return health

FinTech Corp theo dõi streaming health với các metrics sau qua OpenTelemetry:

  • loanbot.stream.ttfb_ms — Time to First Byte histogram (p50, p95, p99)
  • loanbot.stream.tokens_per_second — throughput per connection
  • loanbot.stream.buffer_depth — queue depth (alert nếu >80% capacity)
  • loanbot.stream.dropped_connections — slow clients bị drop (alert nếu >1%)
  • loanbot.stream.active_connections — concurrent streams hiện tại
  • loanbot.stream.tool_call_latency_ms — thời gian CIC query trong stream

Alert threshold: nếu p99 TTFB > 2 giây liên tục 5 phút → scale up API pod; nếu dropped_connections > 5% → investigate network hoặc reduce max_concurrent_streams.

🧠 Kiểm tra nhanh

FinTech Corp muốn cho phép nhân viên pause stream (đang đọc chậm) và resume sau 30 giây. SSE và WebSocket xử lý case này khác nhau như thế nào?

Gợi ý: SSE: Khi client đóng EventSource (pause), server-side connection bị ngắt. Khi resume, client tạo EventSource mới. SSE có Last-Event-ID header — server có thể dùng để resume từ điểm dừng nếu đã lưu event IDs. Cần server-side buffer lưu events đã gửi. WebSocket: Connection persist suốt 30 giây (nếu ping/pong vẫn chạy). Client gửi {"type": "pause"}, server buffer vào queue nhưng không gửi. Khi {"type": "resume"}, drain buffer. Phức tạp hơn nhưng không cần reconnect. Recommendation cho LoanBot: SSE với Last-Event-ID (simpler, stateless server) nếu pause < 60 giây; WebSocket nếu cần persist context dài hơn.

📚 Glossary Module 21

Thuật ngữ Định nghĩa
TTFBTime to First Byte — thời gian từ khi gửi request đến khi nhận byte đầu tiên. Quan trọng hơn total latency cho UX.
Server-Sent Events (SSE)HTTP protocol cho phép server push events tới browser qua persistent connection. Một chiều (server→client).
WebSocketFull-duplex TCP-based protocol — cả client và server có thể gửi messages bất kỳ lúc nào.
gRPC StreamingBinary streaming protocol trên HTTP/2 với Protobuf serialization — hiệu quả cho service-to-service.
StreamingResponseFastAPI class để trả về streaming response từ async generator — tự động set Content-Type và transfer encoding.
EventSourceBrowser native API để consume SSE streams — tự động reconnect, parse event format, dispatch events.
input_json_deltaClaude streaming event type — chứa partial JSON của tool input argument, cần accumulate và parse khi hoàn chỉnh.
BackpressureCơ chế producer chậm lại hoặc dừng khi consumer không theo kịp — tránh buffer overflow và OOM.
asyncio.QueueThread-safe async queue với maxsize — block producer khi đầy (implements backpressure automatically).
Slow ClientClient nhận data chậm hơn tốc độ server generate — cần detect và either throttle hoặc drop gracefully.
Ping/PongWebSocket heartbeat mechanism — server gửi ping mỗi N giây, client reply pong — detect dead connections.
Connection PoolQuản lý nhiều WebSocket connections đồng thời — map session_id → connection, broadcast, selective send.
Last-Event-IDSSE header cho phép client resume stream từ event cuối đã nhận khi reconnect — server replay missed events.
X-Accel-Buffering: noHTTP header để disable nginx buffering — critical cho SSE, nếu không nginx sẽ buffer toàn bộ trước khi gửi.
Token RateSố LLM tokens gửi đến client mỗi giây — metric để detect slow clients và measure streaming performance.

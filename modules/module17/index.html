🔑 Key Insight: Vision > OCR

Traditional OCR chỉ extract text — không hiểu context. Claude Vision hiểu document structure: biết "Net Income" là số cuối cùng trong cột lương, không phải "Tax Deduction". Với payslip có format khác nhau từ hàng nghìn employer khác nhau, rule-based OCR fail 23% cases. Claude Vision generalize tốt hơn vì nó reason về layout, label, và context.

Python (Claude Vision API) 
import anthropic, base64, pathlib

client = anthropic.Anthropic()

def extract_from_document(doc_path: str, doc_type: str) -> dict:
    """Extract structured data từ document image/PDF."""
    # Load và encode document
    raw   = pathlib.Path(doc_path).read_bytes()
    b64   = base64.standard_b64encode(raw).decode("utf-8")
    media = "image/jpeg" if doc_path.endswith(('.jpg','.jpeg')) else "application/pdf"

    # Schema cho từng doc type
    schemas = {
        "payslip": """{"employee_name": str, "employer": str, "id_number": str,
            "period": "YYYY-MM", "gross_income": int, "net_income": int,
            "total_deductions": int, "allowances": int}""",
        "bank_statement": """{"account_number": str, "account_holder": str,
            "period_months": int, "avg_monthly_balance": int,
            "avg_monthly_inflow": int, "avg_monthly_outflow": int,
            "overdraft_incidents": int, "largest_transaction": int}""",
    }

    response = client.messages.create(
        model="claude-sonnet-4-6",  # sonnet for vision tasks
        max_tokens=1024,
        messages=[{
            "role": "user",
            "content": [
                {
                    "type":       "image",
                    "source": {"type": "base64", "media_type": media, "data": b64}
                },
                {
                    "type": "text",
                    "text": f"""Extract data từ {doc_type} này.
Trả về ONLY valid JSON theo schema:
{schemas.get(doc_type, '{}')}

Nếu field không tìm thấy → null. Amounts in VNĐ (integers)."""
                }
            ]
        }]
    )
    import json
    return json.loads(response.content[0].text)
🧠 Kiểm tra nhanh

Tại sao dùng claude-sonnet-4-6 thay vì claude-haiku-4-5 cho vision tasks?

Gợi ý: Vision tasks — đặc biệt document extraction — yêu cầu model phải reason về layout, font, context, và số liệu tài chính phức tạp. Sonnet có better visual understanding, ít hallucinate hơn với blurry scans, và handle các payslip format kỳ lạ tốt hơn. Haiku nhanh hơn và rẻ hơn nhưng accuracy thấp hơn ~8% trên các documents phức tạp. LoanBot dùng haiku cho CCCD verification (structured, always clear), sonnet cho payslip/bank statement (complex, variable format).

⚠️ Cảnh báo: Document Tampering Detection

LoanBot phải detect các documents bị chỉnh sửa. Red flags: font inconsistency trong cùng document, số liệu không hợp lý (salary 200M/tháng nhưng DTI = 0.1%), metadata mismatch giữa PDF creation date và document date, và rounded numbers (income = 20,000,000 đúng VNĐ thay vì 19,800,000). Khi phát hiện → không tự động reject mà trigger HITL Level 2 với reason = "potential_document_tampering".

🏢 Tình huống: FC-2024-004 Bank Statement Analysis

Input: 12-page VCB bank statement PDF (200+ transactions), tháng 7/2025 – 6/2026.
Expected extraction:

  • Avg monthly inflow: 31,250,000 VNĐ (vs declared 32M → discrepancy 2.4% → OK)
  • Salary pattern: regular deposit 28th every month from "FPT Software" → income_source = employment
  • Avg end-of-month balance: 8,400,000 VNĐ → healthy savings
  • Overdraft incidents: 0
  • Largest single transfer: 15,000,000 VNĐ (rent payment — recurring)
  • Red flags: none

Decision: APPROVE — bank statement xác nhận income và spending pattern ổn định.

Python (Bank Statement Extractor) 
from dataclasses import dataclass
from typing import List, Optional

@dataclass
class BankStatementData:
    account_holder:       str
    bank:                 str
    period_months:        int
    avg_monthly_inflow:   int     # VNĐ
    avg_monthly_outflow:  int
    avg_end_balance:      int
    overdraft_count:      int
    income_source:        str     # "employment" | "business" | "mixed" | "unknown"
    income_regularity:    str     # "regular" | "irregular" | "volatile"
    employer_name:        Optional[str] = None
    red_flags:            List[str] = None
    confidence:           float = 0.0

BANK_STATEMENT_PROMPT = """
Đây là sao kê ngân hàng của khách hàng vay vốn tại FinTech Corp.
Phân tích 12 tháng giao dịch và trả về JSON:

{
  "account_holder": string,
  "bank": string,
  "period_months": int,
  "avg_monthly_inflow": int (VNĐ),
  "avg_monthly_outflow": int,
  "avg_end_balance": int,
  "overdraft_count": int,
  "income_source": "employment"|"business"|"mixed"|"unknown",
  "income_regularity": "regular"|"irregular"|"volatile",
  "employer_name": string|null,
  "salary_deposit_day": int|null,
  "largest_transfer_vnd": int,
  "red_flags": [string]
}

Red flags cần detect:
- Overdraft >2 lần/năm
- Gambling keywords: betano, 1xbet, casino, bet
- Unusual round transfers (10M, 20M, 50M) đến unknown recipients
- Income variance >40% between months (volatile)
- Balance consistently near zero before salary

Confidence: tính dựa trên clarity của document (0.0-1.0).
"""

def analyze_bank_statement(pdf_path: str, declared_income: int) -> dict:
    """Full bank statement analysis with cross-check vs declared income."""
    raw_data = extract_from_document(pdf_path, "bank_statement")

    # Cross-check declared income
    disc_pct = abs(raw_data['avg_monthly_inflow'] - declared_income) / declared_income
    raw_data['income_discrepancy_pct'] = round(disc_pct * 100, 1)
    raw_data['income_verified']       = disc_pct < 0.10  # <10% discrepancy → verified

    if disc_pct >= 0.20:
        raw_data['red_flags'] = raw_data.get('red_flags', []) + [
            f"Income discrepancy {disc_pct:.1%} vs declared"
        ]
    return raw_data

Bài 4: Property Document Analysis — Sổ Đỏ, Sổ Hồng & Collateral Verification

🎯 Sau bài này, bạn sẽ:

  • Extract key fields từ Giấy chứng nhận quyền sử dụng đất (Sổ đỏ/Hồng)

  • Detect encumbrances: existing mortgages, liens, legal disputes
  • Flag mismatch giữa property address trong deed và valuation report
📌 Khái niệm: Collateral Verification Chain

Để approve loan với collateral, LoanBot phải verify 4 links trong chain: (1) Identity: người đứng tên sổ = người vay (CCCD match); (2) Encumbrance: sổ không có mortgage/lien đang tồn tại; (3) Valuation: địa chỉ trong sổ = địa chỉ định giá; (4) Area: diện tích trong sổ ≈ diện tích định giá. Nếu bất kỳ link nào fail → HITL Level 2. LoanBot không thể approve secured loan với broken chain.

Python (Property Deed Extractor) 
PROPERTY_DEED_PROMPT = """
Đây là Giấy chứng nhận quyền sử dụng đất (Sổ đỏ/Hồng) Việt Nam.
Extract và trả về JSON:

{
  "certificate_number": string,
  "issue_date": "YYYY-MM-DD",
  "issuing_authority": string,
  "owner_names": [string],
  "owner_ids": [string],
  "parcel_id": string,
  "address": string,
  "area_sqm": float,
  "land_purpose": string,
  "land_use_term_years": int|null,
  "encumbrances": [string],
  "has_existing_mortgage": boolean,
  "signature_present": boolean,
  "confidence": float
}

QUAN TRỌNG:
- encumbrances: list các ghi chú về thế chấp, tranh chấp, hạn chế
- has_existing_mortgage: true nếu có "đã thế chấp" hoặc mortgage stamp
- Nếu không rõ → null, không đoán mò
"""

def verify_collateral_chain(deed_data: dict, application: dict, valuation: dict) -> dict:
    """Verify all 4 links in collateral verification chain."""
    issues = []
    warnings = []

    # Link 1: Identity — owner names must match applicant
    app_name = application['name'].lower()
    deed_owners = [n.lower() for n in deed_data.get('owner_names', [])]
    if not any(app_name in o or o in app_name for o in deed_owners):
        issues.append(f"Identity mismatch: {app_name} not in deed owners {deed_owners}")

    # Link 2: No existing encumbrances
    if deed_data.get('has_existing_mortgage'):
        issues.append("Existing mortgage on property — cannot use as collateral")

    # Link 3: Address consistency
    deed_addr = deed_data.get('address', '').lower()
    val_addr  = valuation.get('address', '').lower()
    if deed_addr and val_addr and deed_addr[:30] != val_addr[:30]:
        warnings.append(f"Address mismatch: deed='{deed_addr[:40]}' vs valuation='{val_addr[:40]}'")

    # Link 4: Area consistency (allow 5% tolerance)
    deed_area = deed_data.get('area_sqm', 0)
    val_area  = valuation.get('area_sqm', 0)
    if deed_area and val_area:
        area_diff = abs(deed_area - val_area) / deed_area
        if area_diff > 0.05:
            warnings.append(f"Area mismatch: deed={deed_area}m² vs valuation={val_area}m² ({area_diff:.1%} diff)")

    return {
        'chain_valid':   len(issues) == 0,
        'hitl_required': len(issues) > 0,
        'critical_issues': issues,
        'warnings':        warnings,
        'hitl_level':     2 if issues else (1 if warnings else None)
    }

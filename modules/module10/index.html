📌 Khái niệm: Token là gì?

LLM providers tính tiền theo token, không phải ký tự hay từ. 1 token ≈ 4 ký tự tiếng Anh ≈ 2–3 ký tự tiếng Việt. Anthropic Claude phân biệt input tokens (prompt + context bạn gửi) và output tokens (response Claude trả về). Output tokens thường đắt hơn input tokens 3×–5×.

FinTech Corp chạy LoanBot v3.0 với claude-haiku-4-5 (tất cả 8 agents). Trung bình mỗi hồ sơ:

  • Input tokens tổng: ~12.000 tokens (system prompt 800 × 8 agents + context)
  • Output tokens tổng: ~2.400 tokens (kết quả mỗi agent ~300 tokens)
  • Chi phí/hồ sơ: (12K × $0.25 + 2.4K × $1.25) / 1.000.000 = $0.006 ≈ 150 VNĐ
  • Chi phí 50.000 hồ sơ/tháng: $300/tháng ≈ 7.500.000 VNĐ
  • Chi phí manual: 50.000 × 350.000 = 17.500.000.000 VNĐ/tháng

ROI: Tiết kiệm 17.492.500.000 VNĐ/tháng — LoanBot tự hoàn vốn trong < 1 ngày.

Công thức tính chi phí mỗi agent call
Cost = (InputTokens × InputPrice + OutputTokens × OutputPrice) / 1,000,000
Tính cho từng agent riêng → sum up → cost per loan application

1.3 Cost Attribution: Ai "tốn tiền" nhất?

Trong Module 8, bạn đã đo latency per agent. Chi phí cũng cần được attributed tương tự — mỗi agent phải "trả lời" cho phần chi phí của mình. Pattern này gọi là Cost Attribution.

Python 
# Cost attribution: gắn chi phí vào từng agent span
def calculate_agent_cost(
    model: str,
    input_tokens: int,
    output_tokens: int
) -> float:
    # Giá mỗi model (USD per token)
    PRICING = {
        "claude-haiku-4-5-20251001":   (0.25e-6, 1.25e-6),
        "claude-sonnet-4-6":           (3.0e-6,  15.0e-6),
        "claude-opus-4-8":             (15.0e-6, 75.0e-6),
    }
    in_price, out_price = PRICING.get(model, (0, 0))
    return input_tokens * in_price + output_tokens * out_price

# LoanBot cost report per hồ sơ
def cost_report(trace: dict) -> dict:
    costs = {}
    for span in trace["spans"]:
        agent = span["agent.name"]
        costs[agent] = calculate_agent_cost(
            span["llm.model"],
            span["llm.tokens.input"],
            span["llm.tokens.output"]
        )
    costs["total"] = sum(costs.values())
    return costs

Bài 2: Caching Strategies — Giảm 60–80% Chi Phí

🎯 Sau bài này, bạn sẽ:

  • Hiểu 3 lớp caching cho LLM systems (Prompt Cache, Semantic Cache, Result Cache)
  • Tính được cache breakeven point và cache hit rate cần thiết
  • Implement Anthropic Prompt Caching với 5-minute TTL
  • Biết khi nào KHÔNG nên cache (fresh data requirements)

Caching là kỹ thuật tiết kiệm chi phí hiệu quả nhất cho LLM agents. Với LoanBot, system prompts của 8 agents gần như không đổi giữa các hồ sơ — đây là cơ hội cache lý tưởng. Anthropic hỗ trợ Prompt Caching với 90% discount cho cached tokens.

2.1 Ba Lớp Caching

L1
Prompt Cache (Provider-side)
Anthropic giữ KV cache của system prompt trong 5 phút. 90% discount trên cached tokens. TTL auto-reset mỗi request.
-90% cost
L2
Semantic Cache (Application-side)
Embed input → tìm trong vector store → nếu similarity > 0.95 → return cached response. Áp dụng cho CIC queries giống nhau.
-30–50%
L3
Result Cache (Business Logic)
Cache kết quả cuối (credit decision) cho cùng customer_id trong 24h. Không cần re-run pipeline nếu re-apply ngay.
-10–20%
💡 Analogy: Nhân viên ngân hàng và bộ nhớ

Hình dung Prompt Cache như nhân viên đã đọc sẵn quy trình trước ca làm việc — không cần đọc lại cho mỗi khách. Semantic Cache như nhớ rằng "ông A và ông B có cùng profile nên quyết định giống nhau". Result Cache như ghi vào sổ: "khách C đã được duyệt sáng nay, không cần thẩm định lại".

2.2 Anthropic Prompt Caching — Cách Implement

Anthropic Prompt Caching hoạt động bằng cách đánh dấu phần nào của prompt nên được cache. Phần được cache phải nằm ở đầu message và ít thay đổi nhất.

Python 
import anthropic

client = anthropic.Anthropic()

# System prompt của CreditAgent — gần như không đổi giữa các hồ sơ
CREDIT_SYSTEM_PROMPT = """Bạn là CreditAgent của LoanBot v3.0...
[800 tokens của instructions, scoring rules, output schema]
Đây là phần KHÔNG ĐỔI giữa các hồ sơ."""

def call_credit_agent_with_cache(loan_application: dict) -> dict:
    response = client.messages.create(
        model="claude-haiku-4-5-20251001",
        max_tokens=500,
        system=[
            {
                "type": "text",
                "text": CREDIT_SYSTEM_PROMPT,
                # Đánh dấu để cache phần này
                "cache_control": {"type": "ephemeral"}
            }
        ],
        messages=[
            {
                "role": "user",
                # Phần này THAY ĐỔI mỗi hồ sơ — không cache
                "content": f"Thẩm định hồ sơ: {loan_application}"
            }
        ]
    )
    # Check cache performance
    usage = response.usage
    cache_hit = usage.cache_read_input_tokens > 0
    print(f"Cache: {'HIT ✅' if cache_hit else 'MISS ❌'} | "
          f"Saved: {usage.cache_read_input_tokens} tokens")
    return response

2.3 Cache Breakeven Analysis

Cache Breakeven Point
Min Cache Hits = Cache Setup Cost / (Normal Cost − Cached Cost)
Ví dụ: Setup cost 800 tokens × $0.25/1M = $0.0002 | Savings/hit = 800 × ($0.25−$0.025)/1M = $0.00018 | Breakeven = 2 hits

Với Anthropic Prompt Caching: breakeven chỉ sau 2 lần hit. Với LoanBot xử lý 50.000 hồ sơ/tháng, TTL 5 phút, throughput ~115 hồ sơ/giờ → cache hit rate ≈ 95% trong giờ cao điểm. Tiết kiệm thực tế: ~60% total cost.

Semantic Cache dùng embedding model để tìm queries giống nhau. Thay vì exact-match key, dùng cosine similarity:

  1. Embed query — Gọi embedding model (nhỏ, rẻ hơn LLM) cho customer profile
  2. Vector search — Tìm trong Redis Vector Store: có cached result nào với similarity > 0.95?
  3. Cache hit — Return cached result, không call LLM
  4. Cache miss — Call LLM, store result + embedding vào cache với TTL 24h

Áp dụng tốt cho CreditAgent (CIC score ít thay đổi trong ngày), không áp dụng cho FraudAgent (real-time fraud signals thay đổi liên tục).

🧠 Kiểm tra nhanh

FraudAgent của LoanBot KHÔNG nên dùng Result Cache (L3). Tại sao?

Gợi ý: FraudAgent check real-time signals: blacklist được update liên tục, velocity patterns thay đổi theo giờ. Nếu cache result 24h, LoanBot có thể approve người vừa bị thêm vào blacklist 1 giờ trước. Fresh data requirement trumps cost savings. L1 Prompt Cache vẫn OK vì chỉ cache system instructions, không cache kết quả.

Python 
class TaskRouter:
    """Route loan tasks đến model phù hợp dựa trên complexity."""

    # Model mặc định cho từng agent
    DEFAULT_MODEL = {
        "AuditAgent":      "claude-haiku-4-5-20251001",
        "ReportAgent":     "claude-haiku-4-5-20251001",
        "CreditAgent":     "claude-haiku-4-5-20251001",
        "IncomeAgent":     "claude-haiku-4-5-20251001",
        "FraudAgent":      "claude-haiku-4-5-20251001",
        "RiskAgent":       "claude-haiku-4-5-20251001",
        "ComplianceAgent": "claude-haiku-4-5-20251001",
    }

    def complexity_score(self, task: dict) -> float:
        # Tính complexity 0.0–1.0 dựa trên đặc điểm hồ sơ
        score = 0.0
        loan_amount = task.get("loan_amount_vnd", 0)
        if loan_amount > 1_000_000_000:   score += 0.4  # > 1 tỷ VNĐ
        elif loan_amount > 500_000_000: score += 0.2  # > 500 triệu
        if task.get("fraud_signals"):     score += 0.2
        if task.get("is_self_employed"):  score += 0.15
        if task.get("foreign_income"):    score += 0.25
        return min(score, 1.0)

    def route(self, agent_name: str, task: dict) -> str:
        score = self.complexity_score(task)
        default = self.DEFAULT_MODEL[agent_name]
        # Chỉ upgrade nếu task phức tạp VÀ agent cần reasoning
        needs_reasoning = agent_name in {"RiskAgent", "ComplianceAgent"}
        if score > 0.5 and needs_reasoning:
            return "claude-sonnet-4-6"  # Upgrade cho hồ sơ phức tạp
        return default

4.1 Parallel Execution — Tại Sao Phase Design Quan Trọng

LoanBot v3.0 đã implement asyncio.gather() trong Module 7. Đây là tối ưu latency — nhưng nó cũng ảnh hưởng đến cost pattern. Parallel calls tạo burst load → có thể trigger rate limiting → retry costs. Sequential giảm burst nhưng tăng latency.

⚠️ Rate Limit = Hidden Cost

Khi 8 agents call Anthropic API đồng thời (asyncio.gather), bạn có thể hit rate limit (tokens per minute). Response: 429 → retry với exponential backoff → thêm latency + wasted compute. Giải pháp: implement token bucket rate limiter trước khi gọi API.

4.2 Anthropic Batch API — 50% Cost Reduction

Anthropic cung cấp Message Batches API: gửi nhiều requests trong 1 batch, Anthropic xử lý async và trả kết quả trong vòng 24 giờ. Đổi lại: 50% discount trên tất cả token costs.

🔑 Key Insight: Batch vs Real-time Strategy

Không phải mọi task đều cần real-time response. LoanBot có thể split:

  • Real-time pipeline (latency-sensitive): Hồ sơ VIP, deadline hôm nay → $0.006/hồ sơ
  • Batch pipeline (overnight): Hồ sơ regular, re-check quarterly, audit reports → $0.003/hồ sơ

30–40% hồ sơ có thể batch → tiết kiệm thêm 15–20% tổng chi phí.

Python 
import anthropic
from datetime import datetime, timedelta

client = anthropic.Anthropic()

def submit_batch_credit_checks(applications: list) -> str:
    """Gửi nhiều hồ sơ sang Batch API — 50% cheaper, kết quả trong 24h."""
    requests = []
    for app in applications:
        requests.append({
            "custom_id": app["loan_id"],  # Track từng hồ sơ
            "params": {
                "model": "claude-haiku-4-5-20251001",
                "max_tokens": 500,
                "messages": [{
                    "role": "user",
                    "content": f"Phân tích tín dụng: {app}"
                }]
            }
        })
    # Submit batch — return batch_id để poll sau
    batch = client.beta.messages.batches.create(requests=requests)
    print(f"Batch submitted: {batch.id} | {len(requests)} applications")
    print(f"Expected completion: {datetime.now() + timedelta(hours=24)}")
    return batch.id

def collect_batch_results(batch_id: str) -> dict:
    # Gọi sau khi batch done (poll hoặc webhook)
    results = {}
    for result in client.beta.messages.batches.results(batch_id):
        loan_id = result.custom_id
        if result.result.type == "succeeded":
            results[loan_id] = result.result.message.content[0].text
    return results

4.3 Mixed Strategy cho LoanBot v3.0

  1. Priority Classification

    Khi nhận hồ sơ: nếu loan > 1 tỷ HOẶC khách VIP HOẶC deadline trong ngày → Real-time pipeline. Còn lại → queue cho batch.

  2. Real-time Pipeline

    asyncio.gather() như Module 7, với Rate Limiter và Prompt Caching. Target: xử lý trong 5 giây.

  3. Batch Queue

    Gom hồ sơ batch trong ngày, submit Batch API lúc 22:00, nhận kết quả 6:00 hôm sau. Cost: 50% of real-time.

  4. Result Merge

    Loan officer nhận notification buổi sáng với kết quả batch đã processed. Real-time kết quả hiển thị ngay khi submit.

5.3 Budget Alerts — Tự động hóa Cost Governance

Python 
from dataclasses import dataclass
from typing import Optional

@dataclass
class BudgetConfig:
    monthly_limit_usd: float = 400.0
    warning_threshold: float = 0.75  # Alert tại 75%
    critical_threshold: float = 0.90  # Alert tại 90%
    per_loan_limit_usd: float = 0.02  # Anomaly nếu 1 hồ sơ > $0.02

class CostGovernor:
    def __init__(self, config: BudgetConfig):
        self.config = config
        self._monthly_spend = 0.0
        self._loan_costs = []

    def record_and_check(self, loan_id: str, cost_usd: float) -> Optional[str]:
        # Ghi nhận chi phí và kiểm tra ngưỡng
        self._monthly_spend += cost_usd
        self._loan_costs.append(cost_usd)
        pct = self._monthly_spend / self.config.monthly_limit_usd

        if cost_usd > self.config.per_loan_limit_usd:
            return f"🚨 ANOMALY [{loan_id}]: ${cost_usd:.4f} > limit ${self.config.per_loan_limit_usd}"
        if pct >= self.config.critical_threshold:
            return f"🚨 CRITICAL: Monthly budget {pct:.0%} used (${self._monthly_spend:.2f})"
        if pct >= self.config.warning_threshold:
            return f"⚠️ WARNING: Monthly budget {pct:.0%} used (${self._monthly_spend:.2f})"
        return None  # Trong budget
